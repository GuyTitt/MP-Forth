Objectif-Final:
- avoir un programme informatique fonctionnant sur microcontroleur
- ce type de programme existe mais mon objectif n'est pas d'utiliser le programme, du moins dans un premier temps mais de le réaliser et comprendre son fonctionnement dans le détail.
Développement du projet
- initialement je voulais réaliser une implémentation du langague forth en nu-metal sur esp32. j'ai vu que le constructeur ne laissait pas cette possibilité si l'on voulait untiliser toutes les ressources de l'ESP32-S3N16R8 sur le kit de développement wroom.
- de plus aucune simulation n'était suffisament performante toute avaient un défaut redibitoire
- je pars donc sur un langage évolué que je connais (micropython) dans lequel j'écris un programme de simulation de forth (style fig forth, eforth, esp32forth, punyforth...) dans une structure la plus proche d'une version assembleur sans utiliser les performances des structures python (collection, dict...) et des libraries ( regex... ). une plage mémoire sera attribué "à la machine python". pour y placer les macros, les data, les piles. 
Python au départ aura les primitives de base , il pourra lire un tableau en mémoire contenant les macros pour augmenter son vocabulaire puis il passera la main à un repl pour obtenir des macros pour réaliser le programme applicatif ou effectuer des tests.
les tests seront fait sur wokwi en ligne puis sur un composant réél sur le composant réél le tableau en ascii disparaitra laissant place à un tableau binaire résultat de la compilation.
- dans un troisième temps tout sera traduit en assembleur et compilé le programme python ne fera que lancer le programme assembleur puisqu'on ne sait pas faire un vrai boot en assembleur pur.
est-ce réaliste ? Peux tu le détailler, le compléter et améliorer ce projet ?
